# Agent Teams Harness - 原始需求文档

> 创建时间: 2026-03-30
> 需求提出: Project Owner
> 状态: 已确认

---

## 1. 项目背景

当前团队使用飞书多维表格进行敏捷管理，希望将需求收集到 MVP 生产的整个流水线全面 Agent Teams 化。

### 1.1 现状
- 敏捷管理工具：飞书多维表格
- 现有表格：
  - 商机跟进表（研发方向）：`https://vcn4qki17ccw.feishu.cn/wiki/Adl6wwiFIiM9DFk1xB7cNgnhnQc`
  - 研发跟进表：`https://ucnxfoj4wtl9.feishu.cn/wiki/T9JJwrXrpiBg4Sk0ke1c9LkhnUf`

### 1.2 目标
将整个研发流水线 Agent 化，每个物理世界角色对应一个 AI 团队，AI 团队可以训练和接入自己的 Agent 或自己的 Claw 完成工作，如果没有专项 Agent，系统提供兜底机制。

## 2. 完整流程描述

### 2.1 一线情报 AI 团队
- **职责**：通过商机跟进表或市场话题热点收集潜在的产品意向
- **输入**：飞书商机跟进表新增记录、市场热点数据
- **输出**：将潜在意向放进敏捷看板意向池
- **触发方式**：飞书自动化触发器（新增行自动触发）

### 2.2 售前分析 AI 团队
- **职责**：结合现有产品能力评估潜在意向的价值
- **评估维度**：
  - 技术可行
  - 商业有价值
  - 客户需求强
- **输出**：评估出优先级，更新敏捷看板意向池的意向优先级状态

### 2.3 AI 产品负责人
- **职责**：基于产品规划方向性、路标、ROI、结果导向，在优先级上确认先做什么再做什么
- **交互方式**：产品和 AI 一起重排序，确认哪些意向进入特性池
- **输出**：确认进入特性池的意向列表

### 2.4 需求分析师 AI 团队
- **职责**：对特性内容进行需求细化
- **工作内容**：
  - 提出明确的 eval 标准（验收标准）
  - 至少提出 5 套产品实现方案
  - 和团队真人产品设计人员一起确认每个需求怎么做
- **输出**：形成 SDD 标准的需求文档

### 2.5 架构师 AI 团队
- **职责**：针对需求文档提供架构和技术设计方案
- **交互方式**：研发工程师和 AI 架构师一起协作定方案
- **输出**：技术设计文档，需求状态变更为"完成技术设计"

### 2.6 编码 AI 团队
- **职责**：对完成技术设计的需求做开发
- **工作方式**：所有代码变更在指定的 Git 仓库以分支形式开发和管理版本
- **输出**：自测可用后需求状态变为"开发完成"
- **版本管理**：代码变更做版本管理，和技术设计文档关联更新，人工检查架构、状态机

### 2.7 测试 AI 团队
- **职责**：基于需求产品和技术设计文档，设计测试用例
- **工作方式**：跑完测试用例的代码
- **输出**：需求状态完成
- **验收**：产品设计人员进行验收

## 3. 关键约束

### 3.1 飞书集成
- 使用飞书开放平台 API（AppID + AppSecret）
- 需要支持飞书自动化触发器（新增行自动触发 AI 处理）
- 需要调研飞书 API 对接方式

### 3.2 部署要求
- 部署在阿里云服务器
- 7x24 小时运行监听飞书事件
- 不需要部署本地模型

### 3.3 Git 管理
- 使用现有的代码 Git 仓库
- 自动提 PR 和审查代码
- 所有代码变更做版本管理
- 和技术设计文档关联更新
- 人工要检查架构、状态机

### 3.4 真人协作
- AI 完成工作后，通过邮件通知真人
- 真人在飞书多维表格中更新状态确认
- 所有协作在同一个空间

### 3.5 Agent 兜底机制
- 每个物理世界角色可以训练和接入自己的 Agent 或 Claw
- 如果没有自己的专项 Agent，系统需要提供兜底机制
